V1
VOLUME
Quelle quantité de données ?
De Go → To → Po → Eo — l'échelle qui définit le Big Data
Exemples concrets :
Facebook génère ~4 Po de données/jour · Twitter : 500M tweets/jour · LHC CERN : 15 Po/an

Enjeu M1 : Le stockage distribué (HDFS, S3, GCS) devient incontournable. Un SGBD relationnel classique ne tient pas à cette échelle.

HDFS Data Lake Sharding Compression
V2
VITESSE
À quelle vitesse arrivent les données ?
Batch (heures) → micro-batch (minutes) → streaming (millisecondes)
Exemples concrets :
Détection de fraude bancaire en <100ms · Flux boursiers en temps réel · IoT capteurs industriels 10k événements/s

Enjeu M1 : Deux paradigmes opposés — batch (MapReduce, Spark) vs stream processing (Kafka, Flink). Le choix impacte la latence, la cohérence et le coût.

Apache Kafka Apache Flink Spark Streaming Lambda arch.
V3
VARIÉTÉ
Sous quelle forme arrivent les données ?
Structuré (SQL) · Semi-structuré (JSON/XML) · Non-structuré (images, texte)
Exemples concrets :
Logs serveur (texte brut) · Tweets (JSON semi-structuré) · IRM médicale (images DICOM) · Transactions SQL (structuré)

Enjeu M1 : 80% des données mondiales sont non-structurées. Les bases NoSQL (MongoDB, Cassandra, Neo4j) et les data lakes remplacent les schémas fixes pour absorber cette diversité.

NoSQL Parquet/Avro Schema-on-read NLP / CV
V4
VÉRACITÉ
Peut-on faire confiance aux données ?
Bruit · Doublons · Valeurs manquantes · Biais de collecte
Exemples concrets :
Capteurs défectueux renvoyant des valeurs aberrantes · Données clients dupliquées après fusion · Biais dans les données d'entraînement IA (ex. biais racial dans les algorithmes de reconnaissance)

Enjeu M1 : "Garbage in, garbage out" — un modèle ML sur données sales donne des prédictions fausses. La qualité des données conditionne tout le pipeline aval.

Data quality ETL / ELT Deduplication Data lineage
V5
VALEUR
Qu'est-ce qu'on en tire concrètement ?
La donnée brute ne vaut rien — c'est l'insight actionnable qui compte
Exemples concrets :
Netflix : réduction du churn de 4% → économie de 1 Md$/an · Amazon : recommandations = 35% du CA · Météo France : modèles sur 10 Po → alertes précises sauvant des vies

Enjeu M1 : Sans valeur métier, les 4 autres V sont sans intérêt. La data science, le ML et la dataviz sont les outils qui transforment la masse de données en décisions.

Machine Learning BI / Dataviz KPI / ROI Data-driven
